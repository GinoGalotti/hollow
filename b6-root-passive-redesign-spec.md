## The Problem

D42 changed Root's Assimilation from adjacent-territory to same-territory (≥2 presence + ≥2 natives + invaders in same territory → convert invaders to natives). This collapsed clean rates to 0% across all encounters (breach 28–54%) because:

1. Arrival territories (A1/A2/A3) have zero natives — Assimilation never fires where invaders land
2. Inner territories start with 2 natives each but invaders kill them before Resolution
3. The ≥2 presence + ≥2 natives + invaders coincidence almost never occurs
4. Root has no card-level tools to grow, move, or protect natives

B2 isolation test R-A confirmed: removing Assimilation entirely produces 0% Clean / 99.2% Weathered. The new mechanic is functionally equivalent to no Assimilation.

## The Design Direction

Split Assimilation into two tiers and make Root a **native-synergy warden** whose identity is "presence grows the forest, the forest fights back."

### Base Assimilation (always active)

At Resolution, each territory with stacked presence ≥ `assimilation_spawn_threshold` (configurable, test both 2 and 3, we believe 3 is correct but need data) spawns 1 native.

- No invader conversion at base level
- Just grows the native population over time
- Teaches the player: "stack presence → forest grows"
- By Tide 4–5 a player who spreads to 3–4 territories with stacked presence will have built a small native army
- These natives absorb invader combat, buy time, and set up the upgrade

Design intent: this is Root's engine. Like Ember has Ash Trail (self-inflicting corruption as fuel), Root has Assimilation (growing natives as infrastructure). Both wardens build toward something — Ember toward threshold damage, Root toward a native army. The natives ARE the payoff, but they're also fragile — they die in combat, creating corruption that Root then has to manage. That's the tension loop:

**Root's loop: Stack presence → grow natives → natives clash with invaders → invaders die but so do some natives and corruption accumulates → cleanse corruption → grow more natives**

This means Root's skill expression is about WHERE to stack presence (which territories get native production), WHEN to cleanse (corruption from dead natives and invader ravaging), and whether to spread thin (more territories producing natives but fewer per territory) vs stack tall (fewer territories but more reliable production).

### Upgraded Assimilation (assimilation_u1, pool passive)

Adds conversion on top of spawning: territories with ≥2 presence + ≥2 natives + invaders convert `floor(min(presence, natives) / 2)` invaders → natives (weakest first, HP = max(1, invader.MaxHp / 2)).

- This is the payoff for building a native army
- By the time the upgrade unlocks (mid-encounter via threshold), the base mechanic has seeded enough natives for conversion to fire
- Conversion is powerful but conditional — you need the three-way coincidence, which the base mechanic now makes achievable

### What this means for Root's future card design (not implementing now, but framing the direction)

If Root becomes the native-synergy warden, future card design has clear directions:
- Cards that scale with native count in a territory (e.g., "deal damage equal to native count")
- Cards that move natives toward invaders (positioning tools)
- Cards that protect natives from combat death (e.g., "natives in this territory are immune this tide")
- Cards that cleanse corruption generated by native/invader clashes
- Bottoms that spawn natives directly (expensive card burn for immediate native placement)

None of this needs to exist for B6. The base Assimilation passive alone should restore enough native mass to make Root functional. But the card design space is rich and worth noting — Root's deck should eventually lean into this identity.

### Provocation stays in pool (do NOT move to base)

`presence_provocation` (natives counter-attack invaders in presence territories) is extremely powerful once natives exist. If base Assimilation grows natives AND Provocation is always active, Root becomes a native-damage machine from Tide 3 onward with zero player investment.

Keep Provocation as a pool passive (player picks it before the run). This makes the player choose: "do I want my natives to fight (Provocation) or do I want other tools?" That's a meaningful run-shaping decision.

Test B6 without moving Provocation first. If clean rates are still too low after the Assimilation fix, THEN consider moving Provocation to base.

## Implementation Spec

### Step 1: Add configurable knob

Add `assimilation_spawn_threshold` to `BalanceConfig`. This controls how much stacked presence is needed per territory for base Assimilation to spawn a native.

- Default: 3 (our starting hypothesis — ≥3 presence in a territory spawns 1 native at Resolution)
- Must be overridable via SimProfile `balance_overrides` so we can A/B test 2 vs 3
- Add to `SIM_REFERENCE.md` under a new "### Natives / Assimilation" section

### Step 2: Implement base Assimilation

In `RootAbility.cs`, change the Resolution logic for Assimilation (when `Gating.IsActive("assimilation")` but NOT upgraded):

```
At Resolution:
  For each territory where root has presence >= assimilation_spawn_threshold:
    Spawn 1 native (default HP from BalanceConfig.default_native_hp)
    Log: "ASSIMILATION: Spawned Native at {territory} (presence={count})"
```

No invader conversion at base level. Just spawn.

### Step 3: Gate conversion behind upgrade

When `Gating.IsActive("assimilation_u1")` (or however the upgrade check works in the current code):

```
At Resolution (runs AFTER base spawn):
  For each territory where:
    - Root has presence >= 2
    - Territory has natives >= 2
    - Territory has invaders
  Convert floor(min(presence, natives) / 2) invaders → natives
    - Target weakest invaders first
    - Converted native HP = max(1, invader.MaxHp / 2)
    - Log: "ASSIMILATION UPGRADE: Converted {invader} → Native at {territory}"
```

This is the existing D42 conversion logic, just gated behind the upgrade flag.

### Step 4: Update SIM_REFERENCE.md

Add under Balance Overrides:

```
### Natives / Assimilation
| Key | Default | Description |
|-----|---------|-------------|
| `assimilation_spawn_threshold` | 3 | Presence needed per territory for base Assimilation to spawn 1 native at Resolution |
```

Create these profiles and run 500 seeds each, Root warden, with B2/B5 marcher counts active:

```

**Profile 2 — Base Assimilation, spawn threshold 2:**

  }

```

Run both profiles on ALL FOUR encounters: standard, scouts, siege, elite.

That's 8 sim runs total (2 thresholds × 4 encounters).

### Step 6: Analyze

Compare against the pre-D42 B5 targets:

| Encounter | B5 Target Clean% | B5 Target Breach% |
|-----------|-------------------|-------------------|
| standard | 52% | 9.6% |
| scouts | 50.6% | 3.2% |
| siege | 43.6% | 5.2% |
| elite | ~30% | ~5% |

Also check:
- Per-tide native count (are natives actually accumulating or dying too fast?)
- Per-tide corruption (does native death create a corruption management challenge?)
- Whether the verbose logs show Assimilation spawning firing regularly from Tide 3 onward
- Whether natives survive long enough to matter (if they all die before next Resolution, the mechanic is too slow)

### Step 7: Decision tree

**If threshold=3 hits B5 targets:** Ship it. Conservative threshold, natives grow slowly, good tension.

**If threshold=3 is too low clean rate but threshold=2 hits targets:** Ship threshold=2. Natives grow faster, more forgiving.

**If threshold=2 is still too low:** The spawn rate is too slow — natives die faster than they're created. Consider:
- Spawning 2 natives instead of 1 per qualifying territory (add `assimilation_spawn_count` knob)
- Lowering `assimilation_spawn_threshold` to 1 (any presence spawns)
- Adding native HP bonus (`default_native_hp: 3` instead of 2)

**If threshold=2 is too HIGH clean rate (>70%):** Natives are snowballing. The critical mass problem Gino flagged. Consider:
- Capping natives per territory (e.g., max 3 natives per territory)
- Making native spawn conditional (only if territory has < N natives already)

**If one threshold works for some encounters but not others:** That's fine — encounter-specific tuning comes from the encounter config (marcher counts, tide count, etc.), not from the Assimilation threshold. Pick the threshold that works best across the majority and tune outlier encounters separately.

## What NOT to change

- Do NOT move Provocation to base (test without it first)
- Do NOT change Network Fear or Network Slow mechanics (cosmetic changes, not load-bearing)
- Do NOT add native-related cards (future phase, not B6 scope)
- Do NOT change encounter configs (board state must stay warden-neutral)
- Do NOT touch Ember balance (B1/B4 are shipped and working)
